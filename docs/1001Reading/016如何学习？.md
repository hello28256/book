# 016如何学习？

**兴趣 = 发动机**（提供动力）成就感 = 燃油，没有燃油：发动机迟早熄火。

<img src="https://oss-pai-y1p7zrgkn9rww3n1nb-cn-shanghai.oss-cn-shanghai.aliyuncs.com/v2-d1dd242ca3348662617b341b085b72bd_1440w.jpg" alt="v2-d1dd242ca3348662617b341b085b72bd_1440w.jpg" />

> 为什么很多人小时候喜欢打游戏？
因为游戏设计了大量即时反馈：打怪—>升级—>获得装备—>更强—>继续打怪
整个过程不断产生：成就感—>兴趣—>行动—>成就感，形成正循环。
而学习最大的问题恰恰是：行动—>看不到结果—>失去兴趣—>停止行动
所以真正厉害的人往往会主动制造反馈。例如：学习编程时：今天写个Hello World、今天写个计算器、今天写个网页，不断获得小成功。
> 

认知—>行动—>反馈—>成就感—>兴趣—>持续行动
这里面最神奇的是：兴趣不一定是起点，很多时候兴趣是结果。
**学习失败最常见的原因，也不是缺兴趣，而是迟迟不行动，因此永远体验不到成长带来的乐趣。**

**行动 = 车轮**（让车真正前进）

<img src="https://oss-pai-y1p7zrgkn9rww3n1nb-cn-shanghai.oss-cn-shanghai.aliyuncs.com/image-20260624104329082.png" alt="image-20260624104329082" />

为什么行动最重要？因为只有行动能创造现实，兴趣不能、目标不能、方法不能、认知不能。它们都只是存在于大脑里的东西，而行动会把大脑里的东西变成现实。

> 比如学编程：
只有兴趣：我对AI很感兴趣，一个月后：还是不会写代码
只有目标：我要成为AI工程师，一年后：目标还在那里
只有方法：我知道费曼学习法、我知道刻意练习、我知道项目驱动，半年后：知识增加了能力没增加
只有行动：今天写100行代码，结果：代码报错，但是：你获得了经验，第二天：再写100行，又报错，但是：你又获得了经验，能力开始增长。
> 

**没有行动**：** 输入—>输入—>输入**，永远停留在信息层面。
**有行动**：** 输入—>行动—>反馈—>修正—>成长**，学习闭环才真正形成。
很多人把学习理解成：**学习=获取知识**，其实不是，真正的学习是：** 学习=改变能力**，而能力不会因为知道而改变，能力只会因为做而改变。

而行动解决的是：**把一切可能性变成现实，它是学习体系里唯一能够产生反馈的东西。**

<img src="https://oss-pai-y1p7zrgkn9rww3n1nb-cn-shanghai.oss-cn-shanghai.aliyuncs.com/e303b968-ec2a-4819-aaa9-360eb6f89a71.png" alt="公开书籍方法 (1).png" />

**没有行动，兴趣是幻想，目标是愿望，方法是理论，认知是想法。有了行动，它们才开始产生价值。因为所有高手，都是在“不太会”的时候开始的，而不是在“准备好了”的时候开始的。**

**坚持 = 车轮的耐久度**（保证不会跑一段就停）

![unnamed.png](https://oss-pai-y1p7zrgkn9rww3n1nb-cn-shanghai.oss-cn-shanghai.aliyuncs.com/unnamed.png)

让行动不半途而废，你可以有动力（兴趣）和方向（目标+方法），但是如果缺乏耐久性，一旦遇到困难、挫折或者枯燥，就可能停下。**坚持就是让“车轮”继续滚动，不管路面多坑、多长、多滑**。产生复利效应，学习本质上是能力的累积。每天的小行动，如果能坚持，最终产生指数级成长。例：每天写 1 小时代码 → 1 个月后积累 30 小时 → 1 年后积累 365 小时 → 能写出完整项目。支持循环反馈，坚持让你有机会获得持续反馈，从失败中学习，调整方法，逐步提升认知。如果半途而废，之前的行动和努力就白费了。

**认知 = 挡风玻璃（** 认知决定你能看多远）

<img src="https://oss-pai-y1p7zrgkn9rww3n1nb-cn-shanghai.oss-cn-shanghai.aliyuncs.com/advanced-interlayer-solutions-windshield.jpg" alt="advanced-interlayer-solutions-windshield.jpg" />

A看到的是：太难了，我不适合。B看到的是：这是我的能力边界，突破后就升级了，两个人的行动可能一样，但最终结果完全不同。**认知** 不仅是知识本身，更重要的是你对**自己理解世界和掌握能力的理解**。** 好认知** = 对学习对象、学习过程和自己能力有准确的判断。** 认知自信** = 相信自己能够学会、理解、解决问题，即便遇到困难也不慌张。

> 如果把学习比作打怪：
**目标** = 要打败的怪物
**方法** = 武器和技能
**兴趣** = 战斗的热情
**行动 + 坚持** = 每一次攻击
**认知** = 你对怪物血量、防御、攻击方式的了解
**认知自信** = 你相信自己用手头的技能和策略能够打败它
没有认知自信，即使有目标、方法、兴趣，也容易退缩或者焦虑。
> 

**好认知带来的力量，** 你不会因为“好像太难了”“我可能学不会”而停滞。你知道困难是正常的，是学习的一部分。**提高效率，** 好认知让你知道什么时候需要深入，什么时候可以快速跳过。**增强行动力** 当你相信自己能够解决问题时，行动就更果断。遇到问题也不会慌，先尝试、再查资料、再调整。**认知自信的来源，** 每解决一个小问题，信心就增加一分；小胜利非常重要，比长时间空想要有效得多。遇到问题时及时调整认知，而不是盲目坚持错误的方法，认知不是盲目自信，而是经过实践验证的自信。知道学习不是一蹴而就，而是反复试错、逐步掌握，这让你不会对困难过度恐慌。

**目标 = 导航目的地**（决定去哪）

<img src="https://oss-pai-y1p7zrgkn9rww3n1nb-cn-shanghai.oss-cn-shanghai.aliyuncs.com/gps-d-227327738.webp" alt="gps-d-227327738.webp" />

**目标的重要性，没有很多人想象得那么高，但也绝对不能没有。**

> **没有目标**
就像开车：发动机正常、车轮正常、导航关闭，结果：一直在跑，但不知道去哪。很多人就是这样：今天学 Python，明天学 AI，后天学区块链，下周学量化交易，很努力，但几年后发现：什么都会一点，什么都不精，因为没有明确方向。
> 

> **目标过于重要**
另一种极端是：先确定人生目标、先确定职业规划、先确定未来十年路线
结果：一直规划，从未出发，这和寻找完美学习方法是一个问题。
> 

> **目标真正的作用**
目标最核心的作用其实只有一个：**过滤。** 例如：你的目标是：成为Java后端工程师，那么：Spring Boot 要学、MySQL 要学、Redis 要学，而：Unity游戏开发、iOS开发、Blender建模，暂时可以放后面。目标帮你决定：什么重要，什么不重要
> 

> 
> 
> 
> **目标可以不断修正**
> 很多人有个误区：目标必须一开始就正确，实际上不是。很多人的成长路径是：目标A—>行动—>发现不适合—>目标B—>继续行动—>发现更喜欢—>目标C
> 最后才找到真正适合自己的方向。所以目标更像当前导航，而不是终身绑定的命运，导航输错了可以重新规划路线，停车不动才到不了目的地。
> 

**目标很重要，但它的作用是“校准方向”；行动更重要，因为它决定你是否真的在路上。** 因为方向错了还能修正，而车一直停在原地，再正确的导航也没有意义。

**方法 = 导航路线**（决定怎么去）。

<img src="https://oss-pai-y1p7zrgkn9rww3n1nb-cn-shanghai.oss-cn-shanghai.aliyuncs.com/1718187635945_sJtRdhk3.png" alt="1718187635945_sJtRdhk3.png" />

多人学习失败，不是因为没有方法，而是因为**过度寻找方法，导致迟迟不开始行动。心理学中有一个类似概念：把准备当成了进步。**

> 收藏了 100 篇「如何学习 Python」、看了 50 个「程序员成长路线图」、对比了 20 个 AI 框架。
结果：学习方法：90分，实际技能：0分
因为真正的学习发生在：写第一行代码、做第一个项目、犯第一个错误、修复第一个Bug，而不是研究学习路线图的时候。
> 

> 很多人其实陷入了一个误区：**先找到最优方法**
再开始学习，但现实往往是：**先开始学习→ 遇到问题→ 再优化方法**，因为你不下水，根本不知道自己会遇到什么问题。实际上，方法很多时候是** 结果，不是前提。**
高手的方法论往往来自：**行动—>踩坑—>总结—>形成方法；**
而不是：**找到方法—>直接成为高手**。
> 

学习可以简化成一个公式：兴趣—>行动—>反馈—>调整—>成长。这里面甚至没有“寻找完美方法”这一步。很多程序员最开始：不懂学习方法、不懂知识体系、不懂什么叫刻意练习，但因为喜欢折腾电脑，喜欢写代码，几年下来依然成长得很快。很多时候，真正阻碍学习的不是不会学，而是总想等自己准备好了再开始。而现实是：**开始之后，你才会慢慢准备好，** 因为方法是在反馈中不断修正出来的。

## 目前中国教育存在的问题

1. **中国教育筛选功能很强。** 从现实角度看，中国的教育体系长期承担着大规模选拔人才的功能。大致路径是：小学、初中、中考、高中/职校、高考、大学，在这个过程中会不断分流。这种模式的优点是：标准化、大规模、相对容易组织。但缺点也很明显：学生较早面临选拔压力、容错空间有限、容易让人把考试成绩等同于学习能力。实际上：考试能力 ≠ 学习能力，很多人在学校成绩一般，但进入社会后学习能力很强。也有人考试成绩很好，但离开考试环境后不知道如何自主学习。
2. **很多人学的是“应付考试”，不是“学习”。** 被要求学习科目，而不是学习自己感兴趣的东西。对于很多学生来说：学习=完成作业=考试得分，久而久之形成一种印象：**学习是一种任务。** 而不是：学习是一种探索世界和提升能力的过程。所以不少人毕业以后会出现一种现象：终于不用学习了，因为他们把学习和考试绑定在了一起。
3. **学习方法被神化了。** 很多学生从小接触到的信息是：学霸有秘诀、状元有秘籍、高分有技巧，于是形成一个潜意识：学习好=掌握了某个秘密方法，但现实往往是：学习好=投入时间+持续练习+不断纠错，方法有帮助，但方法很少是决定性因素。
4. **兴趣在学校环境里经常被压缩。** 学校需要统一教学。统一教学天然会遇到一个问题：**30个人30种兴趣，但课程只有固定几门。** 所以很多人的兴趣并不会在学校里被充分发现。例如：编程、摄影、音乐制作、视频剪辑、设计、创业，这些能力在学校里可能占比并不高，但进入社会后却可能成为职业方向。
5. **不过还有一个容易被忽略的事实。** 虽然教育体系存在很多限制，但成年人之后最大的变化是：**学习权重新回到了自己手里。** 学生时代：学什么别人决定，成年以后：学什么、自己决定，这是非常大的变化。很多人从小接受的是：找到方法—>成绩提高，慢慢发现的是：承认自己不知道—>开始行动—>允许试错—>获得反馈—>不断成长，这其实已经从“应试思维”逐渐转向了“成长思维”。前者关注的是：如何得到正确答案。后者关注的是：如何通过不断尝试，最终找到答案。而在真实世界里，无论是学编程、做产品、创业还是职业发展，后者往往比前者更重要。

**学习最重要的，不是兴趣，不是目标，不是方法，而是：承认不知道，然后敢于开始行动，并且坚持下去。**
